# Issue: 阶段末增加最小 Web 前端验收界面

## User Facing Issue

### Background

本阶段后端目标完成后，需要一个最小 Web 前端来验收内容发布流程，并帮助理解前后端协作。前端不是本阶段深入学习重点，优先由 AI Agent 实现。

### Tasks

- [ ] 提供注册或登录入口。
- [ ] 支持创建文章草稿。
- [ ] 支持查看我的文章。
- [ ] 支持编辑草稿。
- [ ] 支持发布文章。
- [ ] 支持查看公开文章列表和详情。
- [ ] 在 README 中说明如何运行前端或访问页面。

### Acceptance Criteria

- 用户可以通过浏览器跑通注册/登录/创建草稿/编辑/发布/公开查看闭环。
- 前端只消费后端公开 API，不复制后端权限和状态规则。
- 页面简单可用，优先服务验收，不追求复杂设计系统。
- 后端 `go test ./...` 保持通过。
