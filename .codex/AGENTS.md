## A3. handler 层与路由层

### 需要落地的接口

- `POST /register`
- `POST /login`
- `POST /articles`
- `PUT /articles/:id`
- `POST /articles/:id/publish`
- `GET /articles`
- `GET /articles/:id`
- `GET /me/articles`

### 完成标准

- handler 只做请求解析、调用 service、返回响应
- 不在 handler 里写业务判断
- 不在 handler 里写 SQL
- 能区分 request DTO / response DTO 和 model 的职责
- 路由能跑通

### 评估标准

本地启动后，至少能用 curl / Postman 跑通：

- 注册
- 登录
- 创建文章
- 发布文章
- 查询公开文章列表
